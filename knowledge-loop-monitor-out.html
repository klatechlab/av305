แหล่งความรู้ระบบภาพ

LOOP, Monitor Out และการป้องกันภาพวนกลับ

สรุปหน้าที่ของช่อง HDMI LOOP และ HDMI Monitor Out บน NovaStar VX600 พร้อมแนวทางนำ Monitor Out กลับเข้า VAVE Matrix อย่างปลอดภัย และข้อห้ามสำคัญที่อาจทำให้เกิดภาพกระจกซ้อนหรือ Video Feedback Loop

สารบัญ

1. สรุปสั้นที่สุด 2. HDMI LOOP 3. HDMI Monitor Out 4. ตารางเปรียบเทียบ 5. วนกลับเข้า VAVE 6. ภาพกระจกซ้อน 7. Routing ที่ปลอดภัย 8. วิธีแก้เมื่อกดผิด 9. ประโยชน์การใช้งาน 10. แหล่งอ้างอิง

1. สรุปสั้นที่สุด

HDMI LOOP

ภาพตรงจาก Input ที่จับคู่

LOOP ใช้ส่งผ่านภาพต้นฉบับจากพอร์ต Input ที่จับคู่กันโดยตรง ไม่ได้เปลี่ยนตามภาพที่ NovaStar กำลังจัดหรือกำลังแสดงบนจอ LED Wall

HDMI MONITOR OUT

ภาพผลลัพธ์หลังประมวลผล

Monitor Out ใช้แสดงภาพที่สัมพันธ์กับภาพบนจอ LED Wall จึงเปลี่ยนตาม Source, Main Layer, PIP, การสเกล และการจัดตำแหน่งภาพ

ข้อห้ามสำคัญ

หากนำ NovaStar Monitor Out กลับเข้า VAVE Input ช่องหนึ่ง ห้ามเลือก Input ช่องนั้นออกทาง VAVE Output A ในขณะที่ Output A ยังส่งสัญญาณกลับไปยัง NovaStar เพราะจะเกิดวงจรภาพวนกลับเข้าสู่ตัวเอง

2. HDMI LOOP ทำหน้าที่อะไร

บน NovaStar VX600 ช่อง HDMI 1.3-1 รองรับ Loop-through หมายความว่าสัญญาณที่เข้าทาง HDMI 1 จะถูกส่งผ่านออกทางช่อง LOOP เพื่อให้นำไปต่อจอ Monitor หรืออุปกรณ์อื่นได้

แหล่งภาพ → NovaStar HDMI Input 1
HDMI LOOP ของ Input 1
Monitor ตรวจสอบภาพต้นฉบับ

กรณีเพิ่ม Input 2 และ Input 3

LOOP ของ HDMI Input 1 จะไม่เปลี่ยนไปแสดง Input 2 หรือ Input 3 แม้จะสลับภาพหลักของ NovaStar ไปใช้แหล่งภาพอื่น จอที่ต่อกับ LOOP ของ Input 1 จะยังคงเห็นภาพจาก Input 1 เท่านั้น

LOOP เหมาะสำหรับตรวจสอบอะไร

  • แหล่งภาพต้นทางเปิดอยู่หรือไม่
  • สาย HDMI ก่อนเข้า NovaStar ใช้งานได้หรือไม่
  • NovaStar ได้รับภาพจาก Input ที่จับคู่หรือไม่
  • ภาพต้นฉบับมีความละเอียดและสัดส่วนถูกต้องหรือไม่

3. HDMI Monitor Out ทำหน้าที่อะไร

ช่อง HDMI Output ของ VX600 รองรับโหมด Monitor และ Video Output ในโหมด Monitor ภาพที่แสดงบนจอ LED จะถูกประมวลผลให้เหมาะกับ ความละเอียดและเฟรมเรตของช่อง HDMI Output แล้วส่งไปยังจอหรืออุปกรณ์ที่ต่ออยู่

เปลี่ยน Source

ภาพ Monitor Out เปลี่ยนตาม Source ที่กำลังใช้งาน

Main Layer และ PIP

เห็นภาพหลังจัดขนาด ตำแหน่ง และซ้อน Layer แล้ว

Program Feed

ใช้ส่งภาพผลลัพธ์ไปยัง Capture Card, Recorder หรือจอควบคุม

Input 1 / Input 2 / Input 3 และการจัด Layers
NovaStar VX600 ประมวลผลภาพ
Ethernet/LAN → LED Wall
HDMI Monitor Out → Program Feed

หมายเหตุ

ภาพ Monitor Out เป็นภาพที่เหมาะสำหรับใช้ตรวจสอบภาพผลลัพธ์ แต่ไม่จำเป็นต้องตรงแบบพิกเซลต่อพิกเซลกับจอ LED Wall เพราะช่อง HDMI Output อาจปรับภาพให้เหมาะกับความละเอียดของอุปกรณ์ปลายทาง

4. ตารางเปรียบเทียบ LOOP กับ Monitor Out

หัวข้อ HDMI LOOP HDMI Monitor Out
ภาพที่ส่งออก ภาพตรงจาก Input ที่จับคู่ ภาพผลลัพธ์หลังประมวลผล
เปลี่ยนตาม Source บน LED ไม่เปลี่ยน หากยังเป็น LOOP ของ Input เดิม เปลี่ยนตาม Source และ Layout
เห็น PIP และ Layers ไม่เห็นการจัดภาพของ NovaStar เห็นภาพหลังจัด Layers
เหมาะสำหรับ เช็กสัญญาณต้นทาง เช็กภาพสุดท้าย ส่ง Zoom บันทึก หรือถ่ายทอดสด

5. นำ Monitor Out วนกลับเข้า VAVE Matrix ได้หรือไม่

ทำได้ โดยถือว่า NovaStar Monitor Out เป็นแหล่งสัญญาณ HDMI อีกหนึ่งแหล่ง แล้วต่อเข้า Input ที่ว่างของ VAVE Matrix จากนั้นจึงใช้ Matrix ส่ง Program Feed ไปยังปลายทางที่ต้องการ

NovaStar VX600 — HDMI Monitor Out
VAVE Input ที่กำหนดเป็น “VX600 MONITOR RETURN”
VAVE Output B → Monitor / Capture Card / Recorder / ห้องเสริม

ป้ายที่แนะนำให้ติดบน VAVE

VX600 MONITOR RETURN — ห้ามส่งออก OUTPUT A

6. ข้อห้าม: อย่าเลือก Monitor Return ออกทาง Output A

ในระบบปัจจุบัน VAVE Output A ส่งภาพไปยัง Splitter และ NovaStar หากนำ NovaStar Monitor Out กลับเข้า VAVE Input แล้วเลือก Input ดังกล่าวให้ออกทาง Output A ภาพจะถูกส่งกลับเข้า NovaStar ซ้ำอีกครั้ง

VAVE Output A
CYP Splitter → NovaStar VX600
NovaStar HDMI Monitor Out
กลับเข้า VAVE Input “MONITOR RETURN”
กด Input นี้ออก Output A → ภาพวนกลับเข้าสู่ตัวเอง

อาการที่อาจเกิดขึ้น

  • ภาพกระจกซ้อนต่อกันเป็นอุโมงค์
  • ภาพย่อซ้อนเข้าไปเรื่อย ๆ
  • ภาพกะพริบหรือ HDMI จับสัญญาณใหม่ซ้ำ
  • ภาพดำหรือสัญญาณไม่เสถียร

ชื่อทางเทคนิค

อาการนี้เรียกว่า Video Feedback Loop ลักษณะคล้ายการนำกล้องไปถ่ายหน้าจอที่กำลังแสดงภาพจากกล้องตัวนั้นเอง

7. Routing ที่ปลอดภัย

เส้นทางหลัก

Output A ใช้ส่ง Source ปกติไป NovaStar

VAVE Input ปกติ → Output A → Splitter → NovaStar → LED Wall

เส้นทาง Program Feed

Monitor Return ใช้กับ Output B

NovaStar Monitor Out → VAVE Monitor Return → Output B → Capture Card / Monitor / Recorder

กฎจำง่าย

“ภาพที่กลับจาก NovaStar ห้ามย้อนเข้าเส้นทางที่ส่งกลับไป NovaStar”

8. วิธีแก้เมื่อเผลอกดจนเกิดภาพวน

  1. 1. เปลี่ยน VAVE Output A กลับไปยัง Source ปกติทันที

    เช่น Maxhub, HDMI เวที หรือกล้อง PTZ ที่ไม่ใช่ Monitor Return

  2. 2. หากควบคุมไม่ทัน ให้ถอดสาย Monitor Out ที่กลับเข้า VAVE ชั่วคราว

    การตัดเส้นทางย้อนกลับจะหยุด Video Feedback Loop ได้ทันที

  3. 3. รอให้อุปกรณ์ HDMI จับสัญญาณใหม่

    หลังตัดวงจรแล้ว อุปกรณ์อาจใช้เวลาสักครู่ก่อนภาพกลับมาเสถียร

  4. 4. ตรวจสอบ Routing ของ Output A และ B ก่อนต่อสายกลับ

    ยืนยันว่า Monitor Return ถูกส่งไป Output B หรือปลายทางที่ไม่ย้อนกลับเท่านั้น

9. ประโยชน์ของการนำ Monitor Out กลับเข้า Matrix

ส่งเข้า Capture Card และ Zoom

ให้ผู้เข้าร่วมออนไลน์เห็น Program Feed ที่จัดจาก NovaStar

บันทึกหรือถ่ายทอดสด

ส่งไปยัง Recorder, OBS หรือเครื่อง Streaming

จอควบคุมหลังห้อง

เจ้าหน้าที่เห็นภาพผลลัพธ์แม้มองไม่เห็น LED Wall จริง

ห้องเสริมหรือพื้นที่ Overflow

กระจายภาพรายการไปยังอีกห้องหรืออีกพื้นที่หนึ่ง

10. แหล่งอ้างอิงทางเทคนิค

อ้างอิงจาก NovaStar VX600 All-in-One Controller User Manual ซึ่งระบุว่า HDMI 1.3-1 รองรับ Loop-through และช่อง HDMI Output รองรับ Monitor และ Video Output modes โดยในโหมด Monitor จะประมวลผลภาพบนจอ LED ให้เหมาะกับช่อง HDMI Output
